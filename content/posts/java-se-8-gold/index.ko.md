이번에 Java SE 8 Gold를 응시했습니다. Silver를 딴 것이 작년 9월 말이었으니 약 반년 만의 수험입니다. 이전에 Silver를 봤을 때도 [포스트](../java-se-8-silver)로 감상을 남겼기 때문에, 이번에도 Gold를 준비하면서 느낀 점을 정리해 보겠습니다.

## Generics

Generics 문제는 형이 명확하지 않은 상황을 다루는 경우가 많았습니다. 예를 들어 특정 클래스를 상속한 클래스만 인자로 받고 싶을 때 어떻게 해야 하는지 같은 문제입니다. `<? super X>`와 `<? extends X>`의 차이를 정확히 모르면 어려워지는 유형이었습니다.

또 `Comparator<T>`와 `Comparable<T>`도 나와서, 두 개념이 어떻게 다르고 어디에 쓰이는지 이해하지 못하면 풀기 어려웠습니다. 개인적으로는 자주 쓰던 API가 아니어서 처음엔 꽤 고전했습니다. 알고 나면 납득이 가지만, 실제로 앞으로 얼마나 자주 쓰게 될지는 또 다른 문제입니다. 그래도 알아 두면 확실히 유용한 API라 기억해 둘 만했습니다.

## Lambda

Java 1.8의 핵심 기능인 Lambda도 비중이 높았습니다. Lambda는 단순히 메서드를 쉽게 쓴다는 의미만이 아니라, 함수형 인터페이스와 Stream 같은 다른 중요한 API를 함께 이해해야 하는 개념입니다. 그래서 이걸 모르면 Java 1.8을 제대로 이해하지 못했다고 봐도 될 정도입니다.

Lambda 관련 문제는 함수형 인터페이스의 조건, 즉 SAM을 이해하고 있느냐, 그리고 `java.util.function` 패키지의 인터페이스 종류를 알고 있느냐가 중요했습니다. Method Reference 관련 문제도 나왔습니다. Lambda를 충분히 이해하면 결국 문법만 다른 것이라는 걸 알 수 있지만, 그렇지 않으면 코드 자체가 낯설게 느껴질 수 있습니다.

## Stream

Stream 문제도 많이 나왔습니다. Java 1.8에서 추가된 대표 API이기도 하고, 기존 Java API와 성격이 조금 달라서 저처럼 감으로만 쓰던 사람에게는 공부가 많이 됐습니다. 예전에는 `stream()`이나 `collect()` 정도만 썼는데, 이번 시험을 준비하면서 Stream을 더 넓게 활용할 수 있겠다는 생각이 들었습니다.

개인적으로 기존 코드가 문제없이 동작하고 있다면 굳이 Stream으로 바꿀 필요는 없다고 봅니다. 다만 함수형 프로그래밍의 맛을 살짝 느끼게 해 주는 멋진 API라는 점은 분명합니다. 시험 문제도 중간 연산과 종료 연산의 차이, 각 메서드의 용도 같은 것을 묻는 식이라 단순히 메서드 이름만 외우는 문제보다 훨씬 의미 있게 느껴졌습니다. 특히 `peek()`, `sorted()`, `Collectors.groupingBy()` 같은 메서드가 나와서 반가웠습니다.

## I/O

기본적으로는 Java 1.7부터 추가된 NIO 관련 문제가 많았습니다. 이 역시 Stream과 연관해서 `lines()`, `list()`, `walk()` 같은 메서드를 묻는 문제가 자주 나왔습니다. 같은 시기에 등장한 `try-with-resources` 관련 문제도 나와서 좋았습니다.

저는 파일 I/O에서 주로 NIO의 `Files` 클래스를 많이 써서 익숙한 편이었습니다. `close()`를 직접 호출하지 않아도 안전하게 처리할 수 있는 API를 배우는 건 앞으로의 습관에도 좋은 영향을 줄 것 같습니다.

개인적으로는 업무에서 `AutoCloseable`을 구현한 클래스를 만들어 본 적도 있어서 반가웠습니다.

## Thread

동시성 관련 문제도 비중이 꽤 있었습니다. `AtomicInteger`나 `ExecutorService` 같은 중요한 API를 다시 복습할 기회가 됐습니다. 다만 `Runnable`이나 `Callable`까지 다루는 만큼 `CompletableFuture`도 나왔으면 더 좋았겠다는 생각이 들었습니다. `CyclicBarrier`도 나왔는데, 요즘은 자주 쓰이지 않는 느낌입니다. 전체적으로는 `ForkJoinPool`, `CyclicBarrier`, `Runnable`, `Callable`의 시그니처만 잘 기억해도 풀 수 있는 문제가 많았습니다.

## Temporal

`LocalDateTime` 같은 날짜/시간 API 문제도 조금 나왔지만 비중은 낮았습니다. Silver에서도 다뤘던 영역이라 여기서는 포맷터나 예외가 나는 패턴 쪽 문제가 더 많았던 것 같습니다. `ZoneId`도 나왔지만 그렇게 핵심적이진 않은 인상이었습니다.

## Locale

이쪽도 비중은 낮았습니다. 다만 프로퍼티를 어떻게 읽어들이는지는 Locale뿐 아니라 환경 변수를 다룰 때도 필요한 지식이라 기억해 둘 만합니다. API 시그니처를 묻는 문제는 인터넷에서도 쉽게 찾을 수 있으니, 굳이 외우지 않아도 되는 경우가 많다고 느꼈습니다.

## 기타

JDBC 관련 문제도 조금 있었지만, 지금은 자주 쓰이지 않기 때문에 문제 수는 많지 않았습니다. 저는 DB 처리를 할 때 주로 MyBatis나 JPA를 쓰기 때문에 JDBC를 직접 쓰는 일은 많지 않습니다. 그래도 Java에서 DB에 어떻게 접근하는지 기본 구조를 이해하는 정도면 충분하다고 봤기 때문에, 비중은 적절했다고 생각합니다.

Silver까지는 Java API를 얼마나 잘 외우고 있는지를 묻는 느낌이 강했는데, Gold에서는 클래스 설계 문제도 몇 개 나왔습니다. 캡슐화나 싱글톤을 어떻게 구현해야 하는지, `Files.walk()`를 이용한 재귀 처리 같은 문제도 있었습니다. 다만 Gold를 보는 사람이 이런 지식이 없을 것 같지는 않았습니다.

한 가지 아쉬운 점은, 메서드 체이닝과 비슷한 방식으로 사용할 수 있고 Stream에서 요소를 꺼낼 때도 유용한 `Optional`이 거의 언급되지 않았다는 점입니다. Java 1.8의 중요한 기능인데 왜 빠졌는지 조금 의아했습니다.

## 마지막으로

작년에는 Java SE 11 자격도 나왔기 때문에 처음부터 그쪽을 볼까 생각하기도 했습니다. 다만 Oracle 공식 시험 설명을 보면 Java 9에서 도입된 모듈 시스템을 제외하면 시험 내용이 크게 바뀌지 않은 듯합니다.

8도 11도 LTS 버전이므로, 다음 LTS인 17이 나오면 자격도 업데이트될 가능성이 있습니다. 개인적으로는 12부터 14 사이에 들어간 새 기능들이 더 편하다고 느껴서, 이미 8 자격을 가진 사람이라면 11로의 업그레이드 메리트가 아주 크지는 않을 수도 있습니다. 물론 처음 자격을 따는 사람에게는 11이 더 나은 선택일 수 있습니다. 저는 그런 걸 모르고 일단 8을 응시했지만요.

정리하면, 자격 취득 자체도 의미 있었지만 그보다 Java API를 다시 넓게 훑어 볼 수 있었다는 점이 더 만족스러웠습니다. 문제 자체는 패턴이 있는 편이지만, Java 8의 핵심 기능을 한 번 정리해 보고 싶은 사람이라면 응시해 볼 만한 시험이라고 생각합니다.
