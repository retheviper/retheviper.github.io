Python으로 로그 남기기

Jun 15, 2019
4 minute read

코딩에서 로직을 짜는 것만큼이나 중요한 일이 있다면, 내가 쓴 코드가 맞게 동작하는지 확인하는 일이라고 생각합니다. 먼저 명세를 쓰고, 요구사항을 충족하는지 확인하면서 코드를 작성합니다. 하지만 직접 실행해 보기 전에는 코드가 의도대로 움직이는지 알 수 없습니다. 그리고 그 동작이 맞는지 확인하려면, 코드 안에서 다루는 변수나 객체에 올바른 데이터가 들어 있는지 살펴봐야 합니다. Eclipse 같은 IDE에는 디버그 기능이 있어서 중단점을 찍고 흐름을 따라갈 수 있지만, 항상 그런 도구를 쓸 수 있는 것은 아닙니다. 표준 출력으로 확인할 수도 있지만, 실행 후 기록을 남기기 어렵다는 점이 불편합니다.

이럴 때 커스텀 로그를 남기는 방법이 있습니다. 처음부터 로거를 만들어 두면 프로그램이 설계대로 동작하는지 확인하기 쉽고, 완성된 뒤에도 네트워크 장애나 데이터 문제를 추적하는 데 도움이 됩니다. 처음 로그를 남기라는 요구를 받았을 때는 저도 “표준적인 틀이 있나?” 하고 고민했지만, 실제로는 상황에 따라 메시지를 분기하고 그것을 텍스트 파일에 쓰는 구조로 충분했습니다.

이제 실제로 로거를 어떻게 만들었는지 코드를 통해 보겠습니다.

로거 만들기

이번에 만들고 싶은 로거 구조는 간단합니다. 메인 프로그램에서 호출될 때 로그 코드와 작업 코드 같은 값을 받아, 로그 코드에 따라 출력 메시지를 결정합니다. 작업 코드는 필수는 아니지만 로그 코드는 필수로 두고 싶습니다. 또 로그가 작성된 시점을 알 수 있도록 날짜와 시간도 메시지에 넣습니다. 이렇게 하면 로그는 다음과 같은 형태가 됩니다.

2019/06/15 12:00:00 LOG0001: Job code 1234는 정상 종료했습니다.

또 로그 파일 맨 위에는 ---- Job Log ----를 넣어, 이것이 로그 파일임을 표시하고 싶습니다. 이런 식으로 기본 요구사항을 정하고 구현합니다. 제가 작성한 코드를 단순화하면 다음과 같습니다.

# 로그 파일 존재 여부를 확인할 os와 날짜를 남기기 위한 datetime을 import
import os
import datetime

# 로그를 쓰는 함수(job_code는 없을 수도 있으니 기본값을 둔다)
def writeLog(log_code, job_code=''):
    # 로그 파일 경로를 지정(절대 경로)
    log_path = '/log/program_log.log'

    # 로그 코드에 따라 메시지를 결정한다
    if (log_code == 'LOGI001'):
        log_message = 'Job code ' + job_code + '를 실행합니다.'
    elif (log_code == 'LOGI002'):
        log_message = 'Job code ' + job_code + '는 정상 종료했습니다.'
    elif (log_code == 'LOGW001'):
        log_message = 'Job code가 없습니다. 프로그램 시작에 실패했습니다.'
    elif (log_code == 'LOGE001'):
        log_message = 'Job code ' + job_code + '는 비정상 종료했습니다.'
    else:
        log_message = '원인을 특정할 수 없는 오류가 발생했습니다.'

    # 로그 파일이 없으면 로그 파일임을 표시하는 헤더를 먼저 만든다
    if (not os.path.exists(log_path)):
        with open(log_path, mode='a') as log:
            log.write('----- Job Log -----\n')
    else:
        # 로그 파일이 이미 있으면 이어서 기록한다
        with open(log_path, mode='a') as log:
            log.write(datetime.datetime.today().strftime('%Y/%m/%d %H:%M:%S') + ' ' + log_code + ': ' + log_message + '\n')

이제 메인 프로그램에서 로거를 호출하고, 그때 로그 코드와 작업 코드를 넘기기만 하면 됩니다. 로그 종류를 줄이거나 늘리고 싶을 때는 if 문만 수정하면 됩니다.

메인 프로그램에서 로거 호출

메인 프로그램에서는 먼저 로그를 남길 타이밍을 정합니다. 예를 들어 이번에는 DB에 접속하는 작업이 있었기 때문에, DB 접속에 실패했을 때 로거를 호출하도록 했습니다. 그 밖에도 처리 시작이나 종료 시 상태를 남기면 좋습니다.

단순화한 메인 프로그램은 다음과 같습니다. 먼저 만들어 둔 로거를 import하고, 로그를 남기고 싶은 시점에 호출하기만 하면 됩니다. 다만 로거 파일이 메인 프로그램과 다른 경로에 있다면 import 방식이 달라질 수 있으니 주의해야 합니다. 여기서는 메인 프로그램과 로거가 같은 경로에 있다고 가정합니다.

# 단순화한 메인 프로그램
# 종료 코드와 실행 인자를 위해 sys를 import하고, 로그를 남기기 위해 logger의 writeLog를 import
import sys
from logger import writeLog

# 실행 시 커맨드라인 인자를 받기 위한 변수
args = sys.argv

# 함수 예시
def program(job_code):
    # 처리 시작 시 로그를 남긴다
    writeLog('LOGI001', job_code)
    # 정상 처리 로직
    try:
        print('어떤 처리를 수행한다')
        # 정상 종료 로그
        writeLog('LOGI002')
        # 정상 종료 시 종료 코드는 0
        sys.exit(0)
    # 처리 중 문제가 생겼을 때
    except:
        # 비정상 종료 로그
        writeLog('LOGE001')
        # 비정상 종료 시 종료 코드는 9
        sys.exit(9)

# 프로그램 시작 부분
if __name__ == '__main__':
    # 인자(job_code)가 없을 때 검사
    if (args[1] in None):
        # 경고 종료 로그
        writeLog('LOGW001')
        # 경고 종료 시 종료 코드는 1
        sys.exit(1)
    else:
        program(args[1])

마지막으로

어떤 로그를 남기고 싶은지에 따라 로거를 호출하는 위치를 정하면 됩니다. 그리고 인수를 넘기면 어떤 경우에 어떤 로그를 출력할지도 정할 수 있습니다. 구현 자체는 어렵지 않지만, 로거를 분리해 필요할 때만 호출하면 메인 프로그램이 아무리 길어져도 함수 호출만으로 정리할 수 있다는 점에서 효율적입니다. 함수라는 것도 결국 반복을 줄이기 위해 쓰는 것이니까요.

조금 더 찾아보니 Python에는 이미 logging 모듈이 있습니다. 표준 출력에 로그 레벨을 붙여 남길 수 있는 모듈인데, 이를 응용해 파일로 기록하는 쪽이 더 일반적인 접근일 것 같습니다.

결국 중요한 것은 특정 함수 이름을 많이 아는 것보다, 로그를 어떤 구조로 남기고 어떻게 재사용할지를 정하는 일이라고 느꼈습니다. 그런 기준만 잡혀 있으면 구현 자체는 생각보다 단순합니다.
