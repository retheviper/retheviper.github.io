알아 두면 편리한 Linux 팁

May 19, 2019
5 minute read
日本語

초등학교 때 처음 MS-DOS를 만지고, 그 뒤로는 계속 Windows PC만 써 왔기 때문에 저는 Linux를 아주 잘 아는 편은 아닙니다. CLI1는 GUI와는 다른 감각이 있다고 생각했을 정도였습니다. 처음 CentOS를 가상 머신에 설치해 봤을 때도 “아, DOS와 비슷하네” 정도의 인상밖에 없었습니다. 그런데 일을 하면서 직접 써 보니 생각보다 훨씬 편리했습니다. 서버에서 점유율 1위를 차지하는 데는 그만한 이유가 있구나 싶었습니다.

그래서 이번에는 cd나 mkdir 같은 정말 기본적인 명령만 알던 제가 업무를 하면서 배운 팁 몇 가지를 정리해 보려 합니다. 이런 것들은 기본이지만, 실제로 써 보지 않으면 잘 모르는 경우가 많습니다. 명령어와 단축키는 결국 얼마나 잘 쓰느냐가 중요하다고 느꼈기 때문에, 기록 차원에서도 정리해 둡니다.

Tab 키로 자동 완성

GUI보다 CLI가 어렵게 느껴지는 이유는 결국 명령을 직접 입력해야 하기 때문이라고 생각합니다. 파일을 다른 폴더로 옮기는 간단한 작업도 GUI에서는 마우스로 드래그하면 끝나지만, CLI에서는 일일이 명령을 입력해야 합니다. mv 같은 명령을 모르면 시작조차 어렵고, 옵션에 따라 결과도 달라집니다. 무엇보다 경로를 하나씩 입력해야 하는 점이 번거롭습니다.

그런데 이런 불편을 느낀 건 저만이 아니었던 것 같습니다. 예전부터 있었던 기능이지만, Tab 키를 누르면 자동 완성이 됩니다. 긴 경로나 공백이 들어간 이름도 훨씬 쉽게 입력할 수 있습니다. 동작 방식은 대략 이렇습니다.

  1. 파일이나 폴더 이름을 입력한다.
  2. 입력 도중 Tab 키를 누른다.
  3. 이름이 자동으로 완성된다.
  4. 같은 접두어를 가진 항목이 여러 개면 목록이 출력된다.

예를 들어 /home/retheviper/task01 폴더로 이동한다고 해 봅시다.

cd h # 여기서 tab
$ cd home/ # 자동 완성됨
$ cd home/r # 여기서 tab
$ cd home/retheviper/ # 자동 완성됨
$ cd home/retheviper/t # 여기서 tab
task01 task02 task03 # t로 시작하는 폴더가 여러 개라 목록이 출력됨

꽤 편리한 단축키라서 꼭 기억해 두면 좋습니다.

폴더 이동과 목록 출력 명령

현재 폴더의 내용을 확인할 때는 ls나 ll을 많이 씁니다. 이 명령에 경로를 더하면 다른 폴더의 내용도 볼 수 있습니다. 제가 주로 쓰던 방식은 이런 식입니다.

# 현재 폴더 내용 확인
$ ll

# /var 폴더가 있어서 그 안을 확인
$ cd var
$ ll

# 처음 위치보다 위쪽 폴더를 보고 싶을 때
$ cd ..
$ cd ..
$ ll

하지만 경로를 바꾸지 않고도 다른 폴더를 볼 수 있습니다.

# 절대 경로 사용
$ ll /var/lib

# 아래 폴더 보기
$ ll ./lib
$ ll lib

# 위 폴더 보기
$ ll ../

# 더 위 폴더 보기
$ ll ../../

경로가 길어서 이전 위치로 돌아가는 데 시간이 걸릴 때는 이렇게도 할 수 있습니다.

# 긴 경로 안에 있는 상태
$ pwd
/var/lib/jenkins/workspace/job01/git_repository/git
# 또 다른 긴 경로의 폴더로 이동
$ cd /home/retheviper/todo/task01/awesomeblog
$ pwd
/home/retheviper/todo/task01/awesomeblog

# 이전 폴더로 돌아가고 싶을 때
$ cd -
$ pwd
/var/lib/jenkins/workspace/job01/git_repository/git

다만 cd -은 바로 직전 경로와만 오갈 수 있으니 주의해야 합니다. 두 번 연속 입력하면 두 경로를 번갈아 왕복하게 됩니다.

파일 동기화가 가능한 rsync

Linux에서 파일 복사에는 scp를 많이 쓰지만, 저는 rsync가 더 효율적이어서 더 추천하고 싶습니다. rsync는 단순히 파일을 복사하는 것뿐 아니라 폴더와 파일을 동기화할 수 있습니다. 원격 폴더를 동기화할 때도 공개키 인증을 설정해 두면 비밀번호 없이 복사할 수 있는 점이 좋습니다.

동기화가 가능하다는 것은 차이만 찾아낼 수 있다는 뜻이기도 합니다. 이미 대상에 파일이 있으면 원본과 비교해 변경된 부분만 전송하므로 성능도 좋습니다. 원본에서 삭제된 파일을 대상에서도 지울 수 있는 옵션도 있습니다.

# 폴더 동기화
$ rsync origin destination

# 원본에서 삭제된 파일을 대상에서도 삭제
$ rsync --delete origin destination

다른 명령도 마찬가지지만, rsync에서 복사 대상을 지정할 때는 / 유무가 꽤 중요합니다.

예를 들어 원본 폴더를 folder처럼 쓰면 폴더 자체가 복사되고, folder/처럼 쓰면 폴더 안의 내용만 대상이 됩니다. 또 대상 폴더에 권한이 없거나 소유자가 다르면 복사할 수 없으니 주의해야 합니다. 권한은 chmod, 소유자는 chown으로 바꿀 수 있습니다.

동기화할 폴더 중 제외해야 할 폴더가 있다면 --exclude='folder' 옵션으로 뺄 수 있습니다. 이 옵션은 원본과 대상 양쪽에서 적용되므로 꽤 유용합니다.

시스템 사양과 상태 보기

업무에서 성능 테스트를 하거나 특정 작업 중 서버 상태를 모니터링해야 하는 경우가 있었습니다. 먼저 서버 머신 사양부터 보고 싶다면 /proc 아래를 보면 됩니다. 커널이 사용하는 정보가 모여 있습니다.

# CPU 상세 정보 보기
$ less /proc/cpuinfo

# 메모리 상세 정보 보기
$ less /proc/meminfo

실시간으로 CPU나 메모리 사용량을 보고 싶다면 vmstat이 유용합니다.

# 현재 상태 출력
$ vmstat

# 메모리 사용량 출력
$ vmstat -s

# 디스크 활동 출력
$ vmstat -d

# 1초 간격으로 갱신하며 출력
$ vmstat 1

메모리, 디스크, CPU 사용량을 함께 볼 수 있어서 기억해 두면 좋습니다.

긴 출력을 조금씩 보기

ls -al2 같은 명령은 출력이 너무 많아서 한 화면에 다 안 보일 수 있습니다. 이럴 때는 more 명령을 쓰면 출력이 화면을 꽉 채운 뒤 엔터를 눌러 다음 페이지를 볼 수 있습니다.

ls -al | more

이 밖에도 기억해 두면 편리한 명령과 단축키는 정말 많습니다. Linux는 처음에는 낯설어도, 자주 쓰는 몇 가지만 익혀도 작업 속도가 꽤 달라집니다.

저처럼 Linux가 아직 익숙하지 않은 사람에게도 바로 써먹을 수 있는 팁이 되었으면 합니다.

  1. 저는 CUI라는 말에 더 익숙합니다. Character User Interface의 약자인 줄 알았는데, 정확한 이름은 Command Line Interface의 약자인 CLI라고 하네요. ↩︎

  2. ll은 ls -l의 단축형인 것 같습니다. 같은 글자를 두 번 입력하는 것만으로도 대략적인 내용을 보여 주니 편리합니다. 다만 ls -al은 숨김 파일이나 폴더도 함께 보여 줍니다. ↩︎
